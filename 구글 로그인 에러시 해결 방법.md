# 구글 OAuth 로그인 에러 해결 방법

---

## 에러 1: Internal Server Error (최초 증상)

**증상**
```
Internal Server Error
The server encountered an internal error and was unable to complete your request.
Either the server is overloaded or there is an error in the application.
```

**원인**
- `flask-session` 라이브러리가 Python 3.14와 호환 문제로 세션 미들웨어 레벨에서 충돌
- 라우트 핸들러의 `try/except`에도 걸리지 않는 위치에서 발생

**해결**
- `flask-session` 파일시스템 세션 제거
- `google-auth-oauthlib`의 `Flow.fetch_token()` 대신 `urllib`로 직접 토큰 교환

---

## 에러 2: 테스트 return 문이 실제 코드를 막음

**증상**
- 브라우저에 `콜백 도착` 메시지만 뜨고 로그인이 완료되지 않음

**원인**
```python
@app.route("/oauth/callback")
def oauth_callback():
    return f"<h1>콜백 도착</h1>..."  # ← 이 줄이 아래 실제 코드를 막음
    try:
        ...  # 실행되지 않음
```

**해결**
- 테스트용 `return` 문 제거

---

## 에러 3: HTTP 400 Bad Request — Missing code verifier (PKCE 누락)

**증상**
```
토큰 교환 실패 (400):
{
  "error": "invalid_grant",
  "error_description": "Missing code verifier."
}
```

**원인**
- `google-auth-oauthlib`의 `Flow.authorization_url()`이 자동으로 PKCE(`code_challenge`) 포함하여 Google에 요청
- 하지만 토큰 교환 시 `urllib`로 직접 요청할 때 `code_verifier`를 포함하지 않아 Google이 거부

**해결**

1. 로그인 시 `code_verifier`를 세션에 저장:
```python
auth_url, state = flow.authorization_url(...)
session["oauth_state"] = state
session["oauth_code_verifier"] = getattr(flow, "code_verifier", None)  # 추가
```

2. 콜백에서 토큰 교환 시 `code_verifier` 포함:
```python
code_verifier = session.get("oauth_code_verifier")
token_params = {
    "code": code,
    "client_id": GOOGLE_CLIENT_ID,
    "client_secret": GOOGLE_CLIENT_SECRET,
    "redirect_uri": REDIRECT_URI,
    "grant_type": "authorization_code",
}
if code_verifier:
    token_params["code_verifier"] = code_verifier  # 추가
```

## 기타 참고사항

### 테스트 사용자 추가 방법 (OAuth 동의 화면)
1. [Google Cloud Console](https://console.cloud.google.com) 접속
2. 좌측 메뉴 → **"브랜딩"** 클릭 (구 "OAuth 동의 화면")
3. 스크롤 내려서 **"테스트 사용자"** 섹션 → **"+ ADD USERS"** 클릭
4. 이메일 입력 후 저장
> ※ "인증 센터" 메뉴가 아닌 **"브랜딩"** 메뉴에서 설정해야 함

### 서버 완전 종료 방법
- 터미널 창에서 `Ctrl+C`
- 또는 작업 관리자(`Ctrl+Shift+Esc`) → 세부 정보 탭 → `python.exe` 우클릭 → **작업 끝내기**
- 코드 수정 후에는 반드시 서버를 재시작해야 변경사항이 반영됨

### 에러 디버깅 팁
- `urllib.HTTPError`의 실제 응답 내용 출력:
```python
except urllib.request.HTTPError as http_err:
    body = http_err.read().decode()
    print(f"[토큰 교환 오류] {http_err.code}: {body}")
    return f"<pre>토큰 교환 실패 ({http_err.code}):\n{body}</pre>", 400
```
